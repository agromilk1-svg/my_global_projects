# iOS 应用脱壳与导出原理说明

本文档详细介绍了 ECMAIN 中“脱壳并导出”功能的实现原理与技术细节。

## 1. 核心原理概述

iOS App Store 下载的应用通常包含 DRM 保护（FairPlay 加密）。当用户启动应用时，iOS 内核（XNU）的 Mach-O Loader 会读取 `LC_ENCRYPTION_INFO` 加载命令，并利用硬件密钥即时（On-the-fly）解密可执行文件的加密段（TEXT 段的一部分）。

**脱壳（Decryption/Dump）** 的核心思想是：利用操作系统已经完成的解密工作，将内存中**已解密**的数据读取出来，覆盖回磁盘上的加密文件中，并标记为“未加密”。

## 2. 详细实现流程

整个流程分为三个主要阶段：注入、转储（Dump）、重打包。

### 第一阶段：动态库注入 (Injection)
**负责模块**: `ECAppListViewController.m`

为了在目标进程内部执行代码读取内存，我们需要将自定义代码注入到目标 App 中。我们使用 `DYLD_INSERT_LIBRARIES` 环境变量来实现这一点。

1.  **准备 Dylib**: 
    - 主应用将内嵌的 `dump.dylib`（以 Base64 形式存储在 `ECDumpBinary.h`）解码并写入到 `NSTemporaryDirectory()`。
2.  **启动进程**:
    - 使用 `posix_spawn` 函数启动目标 App 的可执行文件。
    - 在环境变量参数 (`envp`) 中设置 `DYLD_INSERT_LIBRARIES=/path/to/dump.dylib`。
    - App 启动时，动态链接器 (`dyld`) 会强制加载我们的 `dump.dylib`。

### 第二阶段：内存转储 (Memory Dump)
**负责模块**: `dump.c` (`dump.dylib`)

当 `dump.dylib` 被加载时，其构造函数 (`__attribute__((constructor))`) 会自动执行，接管控制流并执行以下操作：

1.  **定位 Mach-O Header**:
    - 使用 `_dyld_get_image_header(0)` 获取主程序在内存中的加载基址 (Image Base Address)。
2.  **读取原始文件**:
    - 使用 `_dyld_get_image_name(0)` 获取可执行文件在磁盘上的路径。
    - 将整个文件读取到内存缓冲区 (`file_buf`)。
3.  **解析加密信息**:
    - 遍历 Mach-O Load Commands，寻找 `LC_ENCRYPTION_INFO_64` 命令。
    - 获取加密偏移 (`cryptoff`)、加密大小 (`cryptsize`) 和加密状态 (`cryptid`)。
4.  **数据还原 (关键步骤)**:
    - 如果 `cryptid == 1`（表示已加密），则执行还原。
    - **内存地址计算**: `decrypted_ptr = memory_base + cryptoff`。该地址指向内存中**已经被内核解密**的代码段。
    - **覆盖数据**: 使用 `memcpy` 将内存中的解密数据复制到 `file_buf` 的 `cryptoff` 偏移处，覆盖原有的加密数据。
5.  **移除加密标记**:
    - 将 `LC_ENCRYPTION_INFO_64` 中的 `cryptid` 字段修改为 `0`，告知系统及后续工具该二进制已无 DRM 保护。
6.  **写入中间文件**:
    - 将处理后的 `file_buf` 写入到 App 沙盒的 `Documents/decrypted.bin` 文件中。
    - 调用 `exit(0)` 终止目标进程（防止 App 继续运行干扰用户）。

### 第三阶段：重打包 (Repackaging)
**负责模块**: `ECAppListViewController.m`

主应用 (`ECMAIN`) 监控到 `decrypted.bin` 生成并确认目标进程退出后，开始打包 IPA：

1.  **构建 Payload**:
    - 创建临时目录结构 `Payload/Target.app`。
    - 将目标 App 的所有资源文件（Info.plist, nibs, 图片等）复制到 `Payload` 目录。
2.  **替换二进制**:
    - 删除原有的加密二进制文件。
    - 将 `decrypted.bin` 移动并重命名为 App 的可执行文件名，并赋予可执行权限 (`chmod 755`)。
3.  **压缩导出**:
    - 使用 `ZipWriter` 将 `Payload` 目录递归压缩为 ZIP 格式文件。
    - 将扩展名命名为 `.ipa`（例如 `AppName_Decrypted.ipa`）。
    - 保存到 ECMAIN 的 `Documents/Export/` 目录下供用户分享或安装。

## 3. 技术优势

与传统的 SSH 命令行工具（如 clutch, frida-ios-dump）相比，本方案的优势在于：

1.  **零依赖**: 无需电脑，无需 SSH，无需越狱商店环境，完全在设备端完成。
2.  **沙盒穿透**: 利用 TrollStore 的权限或 App Group 机制（视具体实现），实现了主App与目标App之间的文件交互。
3.  **自动化集成**: 通过内嵌 dylib 和自动化构建脚本，用户不仅无需手动配置环境，开发者编译项目也更加简便。

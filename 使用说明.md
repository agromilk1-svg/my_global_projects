# 设备信息伪装功能使用说明

本文档将指导您如何使用 ECMAIN 的设备信息伪装功能。该功能允许您修改设备参数（如型号、系统版本、UDID 等），并在安装应用时自动将这些伪造信息注入到目标应用中。

## ⚠️ 重要提示

**本功能需要重新编译生效！**
由于刚刚更新了核心安装逻辑，您需要重新运行编译脚本以生成包含自动注入功能的最新 IPA：
```bash
sudo python3 build_full.py
```

## 1. 功能简介

App 使用动态库注入技术（Method Swizzling + fishhook）拦截系统 API 调用，返回您预设的伪造数据。

### 支持伪造的参数
- **系统信息**：iOS 版本、构建版本、内核版本、系统名称
- **硬件型号**：设备型号（如 iPhone15,3）、设备名
- **屏幕参数**：分辨率（宽/高）、缩放比例、原生分辨率、刷新率
- **区域设置**：国家/地区代码、语言代码、时区、货币代码
- **网络信息**：运营商名称、MNC/MCC、网络类型（5G/LTE/Wi-Fi）
- **唯一标识**：IDFA（广告标识符）、IDFV（供应商标识符）
- **硬件指纹**：CPU 核心数、物理内存、磁盘空间

## 2. 如何使用

### 第一步：配置伪造参数
1. 打开 ECMAIN 应用。
2. 切换到底部的 **"设备信息"** 标签页。
3. 您会看到当前设备的真实参数列表。
4. 点击任意想要修改的项目（例如 "设备型号标识"）。
5. 在弹出的对话框中输入想要伪造的值（例如将 `iPhone12,1` 改为 `iPhone15,3`）。
6. 修改完成后，点击右上角的 **"保存修改"** 按钮。
   - 配置将保存到全局路径 `/var/mobile/Documents/device_spoof_config.plist`。

### 第二步：安装应用（自动注入）
配置保存后，**所有通过 ECMAIN 安装的新应用**都会自动应用这些伪装。

1. 在 ECMAIN 中点击 "+" 号或通过 "打开方式" 导入 IPA 文件。
2. 执行安装。
3. 安装过程中，ECMAIN 会自动将 `ECDeviceSpoof.dylib` 注入到目标 APP 的二进制文件中。

### 第三步：验证效果
1. 启动刚刚安装的目标应用。
2. 如果该应用读取设备信息（如设备型号），它将获取到您设置的伪造值。

## 3. 常见问题

**Q: 对已经安装的应用有效吗？**
A: 目前仅对**新安装**（或重新安装）的应用生效。这是因为注入操作是在安装阶段修改二进制文件完成的。如果您想对已安装的应用生效，请在 ECMAIN 中卸载并重新安装它。

**Q: 伪造 UDID/序列号有效吗？**
A: 
- **IDFA/IDFV**：完全有效，支持伪造。
- **UDID/IMEI/序列号**：这些是通过 MobileGestalt 获取的底层硬件信息。普通 App 无法直接读取这些信息（除非利用漏洞）。我们的 Hook 主要是针对普通 App 能够调用的 API。对于越狱检测或高级指纹，效果可能有限。

**Q: 如何恢复真实信息？**
A: 
- 方法 1：在 "设备信息" 页面点击右上角的 **"重置"** 按钮，然后重新安装目标 App。
- 方法 2：在 "设备信息" 页面清空所有修改并保存，然后重新安装目标 App。

**Q: 为什么有些应用闪退？**
A: 注入 Dylib 可能会导致某些有完整性校验的应用闪退。如果遇到这种情况，请尝试先清除伪造配置再安装。
